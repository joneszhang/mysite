Lex (2013.10.01 - 2013.12.30)

This is the curriculum design for The Principle of Compile. It's a lex that can handle a group of regular expressions and finally generate a .cpp file that can work as a DFA. The specific requirements of this project is as follow.

1)Programming based on FA
1.a)Define some REs by yourself
1.b)Convert REs into NFAs
1.c)Merge these NFAs into a single NFA
1.d)Convert the NFA into a DFA with minimum states
1.e)Programming based on the DFA
2)Generating programs based on your own Lex b)Generating programs based on your own Lex

NFA development

This is the most basic and difficult problem, as a programmer with some experience, I come up with lined list and recursion function immediately. But the problem is how to explode the re and how to descend the states to the scions of a father function. And I solve this problem by do analyzing first. I extract the sub string when meeting ‘|’,cause it has the highest priority when other symbols are of the same level. And if there is no ‘|’, The ‘(’ and ‘)’ will be of the first. I extract the sub string of () and ()* and build add states in the rules of the textbook and then descending the two parent states to the sub-function to solve the sub- string

DFA Development

I have to admit that this problem is the most complicated one of all. It contains 5 layers of circulation if you use the method of iteration. At first I decide to use ArrayList object to restore every set of states. But I find that it will be extremely difficult to search the whole diagram. So I choose an indirect way. I use two matched array to store the group number of every state. Once the group is changed, what I need to do is just change the mark on the state, but not move it into a real ArrayList. And I can search the diagram easily. That’s really an interesting improvement.
